| 场景           | `extends` 行为                  | `implements` 行为              |
| ------------ | ----------------------------- | ---------------------------- |
| 类之间          | 单继承，获取父类的实现                   | 不适用（类不能 `implements` 另一个类）   |
| 接口之间         | 多扩展，合并父接口成员                   | 不适用（接口不能 `implements` 任何结构）  |
| `type` 之间    | 通过条件类型或交叉类型扩展                 | 不适用（`type` 不能被 `implements`） |
| 类与接口 /`type` | 不适用（类不能 `extends` 接口 /`type`） | 类必须实现接口 /`type` 的所有成员        |
| 核心目的         | 扩展现有结构（继承实现或合并声明）             | 强制类满足接口 /`type` 定义的契约        |

简单来说：
- `extends` 是 "扩展"，用于继承或合并现有结构（类、接口、`type` 均可参与，但规则不同）
- `implements` 是 "实现"，仅用于类遵守接口或 `type` 定义的规范（必须完整实现所有成员）
