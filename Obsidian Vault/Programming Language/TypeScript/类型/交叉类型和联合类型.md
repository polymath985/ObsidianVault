| 特性   | 联合类型（Union Types）             | 交叉类型（Intersection Types）                 |
| ---- | ----------------------------- | ---------------------------------------- |
| 语法   | 用 \| 分隔类型（如 A\|B）             | 用 & 分隔类型（如 A&B）                          |
| 含义   | 值是多种类型里的任意一种                  | 值同时有多种类型的所有特征                            |
| 逻辑关系 | “或” 关系（A 或 B 或更多）             | “且” 关系（A 且 B 且更多）                        |
| 类型范围 | 组成类型的并集（范围大）                  | 组成类型的交集（范围小）                             |
| 属性访问 | 只能访问所有类型共有的属性 / 方法            | 能访问所有类型的全部属性 / 方法                        |
| 赋值规则 | 符合任意一个类型就能赋值                  | 必须符合所有类型才能赋值                             |
| 典型场景 | 函数参数支持多类型（如输入是字符串或数字）         | 合并接口特征（如用户信息 + 权限信息）                     |
| 注意事项 | 1. 要用类型判断（比如 typeof）才能访问独特属性  | 1. 基本类型交叉（如数字 & 字符串）结果是 never            |
|      | 2. 字面量类型（如 "a"\|"b"）会自动收窄为具体值 | 2. 冲突属性交叉（如 {id: 数字}&{id: 字符串}）结果是 never |
|      | 3. never 和任何类型联合还是原来的类型       | 3. 接口交叉会合并属性，冲突属性变成 never                |
在 TypeScript 中，接口的`extends`（继承）和交叉类型（`&`）都可以实现类型的组合，但它们的行为和适用场景有显著区别，主要体现在**冲突处理**、**适用范围**和**类型关系**上。

  

| **区别维度** | **接口继承（extends）**                                       | **交叉类型（&）**                                                                  |
| -------- | ------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **核心目的** | 扩展接口，建立 “子类型 - 父类型” 的继承关系，子接口拥有父接口的所有成员并可添加新成员。         | 合并多个类型（接口、基本类型、联合类型等），生成一个包含所有类型成员的新类型。                                      |
| **冲突处理** | 若子接口与父接口有**同名属性且类型不同**，会直接报错（不允许冲突）。                    | 若合并的类型有**同名属性且类型不同**，属性类型会被交叉（如`{a: number}` & `{a: string}`的`a`会变为`never`）。 |
| **适用范围** | 仅适用于**接口之间**的扩展（不能继承基本类型、联合类型等）。                        | 可用于**任何类型**的组合（接口、类型别名、基本类型、联合类型等均可）。                                        |
| **类型关系** | 子接口是父接口的**子类型**（`Child extends Parent`为`true`），可赋值给父接口。 | 交叉类型`A & B`既不是`A`的子类型也不是`B`的子类型（除非`A`和`B`有包含关系）。                             |
| **方法处理** | 若父接口有方法，子接口重写时需**兼容父接口的方法签名**（参数可多不可少，返回值需兼容）。          | 若合并的类型有同名方法，方法类型会被交叉（需同时满足所有方法的签名，可能导致函数类型冲突）。                               |
